# PR-173 Kick coverage audit

## Summary

This PR records the current Kick coverage problem before further registry work continues.

## Why

Kick pages can look thin because Kick currently uses seed-list coverage, while Twitch uses directory-style live stream collection.

The current Kick collector has these limits:

```text
MAX_CHANNEL_SLUGS = 220
COLLECT_ATTEMPT_SLUGS = 75
PINNED_ATTEMPT_SLUGS = 20
FETCH_BATCH_SIZE = 10
```

So each run can only observe live channels inside the attempted seed set.

## What was added

- `docs/kick-coverage-audit.md`

## Key decision

Do not treat `kick_channels` registry import as the complete fix.

The registry is required, but Kick also needs candidate expansion and a feedback loop before the pages can approach Twitch-level coverage quality.
