# PR-179 Kick registry feedback loop

## Summary

This PR adds the first registry feedback loop to `collector-kick`.

When collector target source is `registry`, observed and missed attempted slugs update `kick_channels`.

## What changed

- Added `applyRegistryFeedback()` to `workers/collector-kick/src/index.ts`.
- Observed live slugs update live/check metadata, promote to `active`, increase `success_count`, reduce `failure_count`, and raise priority.
- Missed slugs update `last_checked_at`, increase `failure_count`, reduce priority, and move to `cooldown` after repeated misses.
- Added `registryFeedback` metadata to collection results and snapshot metadata.
- Added `docs/kick-registry-feedback-loop.md`.

## Safety

- Feedback only runs when target source is `registry`.
- Seed-list fallback does not mutate `kick_channels`.
- Feedback is best-effort; collection and snapshot writing continue if feedback update fails.
- Blocked/dead rows are not promoted or demoted by this loop.

## Scope

This PR does not:

- add discovery;
- add curated import tooling;
- remove seed-list fallback;
- claim Kick parity;
- claim directory coverage;
- finalize scoring permanently.
