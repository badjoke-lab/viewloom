# PR-185 Add safer Kick live probe modes

## Summary

This PR adds lower-concurrency Kick live probe modes for routine operation when Kick public endpoint errors are high.

## What changed

- Added `kick:live-probe:safe`
  - `--concurrency=4`
  - `--timeout-ms=10000`
- Added `kick:live-probe:slow`
  - `--concurrency=2`
  - `--timeout-ms=15000`
- Kept `kick:live-probe:fast` and `kick:live-probe:aggressive`.
- Updated collector-kick script README with mode guidance.

## Why

Production verification showed the fast mode can still produce a high error count. Safe and slow modes provide lower-load operator options without changing runtime collector behavior.

## Scope

Package script and README changes only.

This PR does not:

- change collector runtime;
- execute D1 writes;
- add discovery;
- claim Kick parity;
- claim directory coverage.
