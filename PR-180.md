# PR-180 Kick status registry feedback

## Summary

This PR surfaces registry target and feedback state in `/api/kick-status`.

## What changed

- `/api/kick-status` now reads latest snapshot `collectorMeta.coverageMode` and `collectorMeta.targetSource`.
- It exposes `registryCandidateCount`, `registryError`, and `registryFeedback` at the top level and under `collector`.
- It adds `coverageMode` and `targetSource` to `latestSnapshot`.
- It updates coverage copy so registry means registry-backed candidate coverage, not directory coverage.
- Added `docs/kick-status-registry-feedback.md`.

## Scope

This PR does not:

- run migration;
- import seed rows;
- add discovery;
- remove seed-list fallback;
- claim Kick parity;
- claim directory coverage.

## Next

After deploy, verify `/api/kick-status` with both seed-list fallback and registry-backed snapshots.
