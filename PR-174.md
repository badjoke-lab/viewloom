# PR-174 Twitch and Kick coverage comparison

## Summary

This PR records the coverage difference between Twitch and Kick before more Kick registry work continues.

## Why

Kick can have a working collector and still produce weak product pages if the observed candidate set is too small.

Twitch is treated as the baseline because it uses directory-style live stream collection.
Kick is still seed-list candidate polling.

## What was added

- `docs/kick-twitch-coverage-comparison.md`

## Key points

- Twitch and Kick must not be treated as equivalent data surfaces yet.
- A fresh Kick snapshot does not mean Twitch-like coverage.
- Status copy must distinguish collector health, freshness, and coverage quality.
- Registry work improves candidate management, but it is not discovery by itself.

## Scope

Docs only.

This PR does not:

- run D1 migrations;
- change collector runtime;
- add discovery;
- change feature rendering;
- claim Kick is fixed;
- claim Kick is Twitch parity.
