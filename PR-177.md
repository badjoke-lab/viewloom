# PR-177 Kick registry verification SQL

## Summary

This PR adds read-only verification SQL and documentation for checking `kick_channels` after operator-controlled migration and seed import.

## What was added

- `workers/collector-kick/sql/verify-kick-channels.sql`
- `docs/kick-registry-verification.md`

## Why

Before changing collector runtime selection, ViewLoom needs a repeatable way to confirm that:

- `kick_channels` exists;
- imported candidate rows exist;
- source/status groups look intentional;
- candidate pool is non-empty;
- top priority rows look valid.

## Scope

Read-only verification only.

This PR does not:

- run D1 migration;
- execute seed import;
- change collector runtime;
- change API status behavior;
- add discovery;
- claim Kick is fixed;
- claim Kick is Twitch parity.

## Next

After operator-controlled migration/import and verification pass, the next implementation PR can make collector-kick select attempted slugs from `kick_channels`.
