# PR-178 collector-kick registry target selection

## Summary

This PR makes `collector-kick` try `kick_channels` as its target source when the registry exists and has usable candidates.

If the registry is missing, empty, or errors, the collector falls back to the existing seed-list behavior.

## What changed

- Added registry target selection in `workers/collector-kick/src/index.ts`.
- Preserved seed-list fallback.
- Added collector metadata for:
  - `targetSource`
  - `coverageMode`
  - `registryCandidateCount`
  - `registryError`
  - `configuredChannels`
  - `attemptedChannels`
- Added `docs/kick-registry-collector-targets.md`.

## Safety

This PR does not require production migration to already exist.

If `kick_channels` is absent, the collector continues with seed-list coverage.

## Scope

This PR does not:

- run D1 migration;
- import seed rows;
- add discovery;
- add curated import tooling;
- update registry feedback fields;
- claim Kick parity;
- remove seed-list fallback.
