# PR-184 Stabilize Kick live probe fast mode

## Summary

This PR changes the default fast live-probe script to a more stable configuration.

## What changed

- `kick:live-probe:fast` now uses:
  - `--concurrency=8`
  - `--timeout-ms=5000`
- The previous high-concurrency setting is preserved as:
  - `kick:live-probe:aggressive`

## Why

Manual verification showed the previous fast mode completed, but produced a very high error count. The lower-concurrency version should be better for routine operator use.

## Scope

Package script change only.

No runtime changes, no D1 writes, no parity claims.
