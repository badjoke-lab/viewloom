# PR-182 Kick live candidate probe script

## Summary

This PR adds a script that probes Kick candidate slugs and writes SQL to boost currently live candidates in `kick_channels`.

## What changed

- Added `workers/collector-kick/scripts/probe-kick-live-candidates.mjs`.
- Updated `workers/collector-kick/scripts/README.md`.

## Why

Registry selection and feedback are working, but latest Kick page density depends on live candidate quality.

Manual testing showed that boosting currently live candidates can raise the latest snapshot from 1-2 observed streams to 4 observed streams.

## Scope

The script only generates reviewable SQL.

It does not:

- execute D1 writes automatically;
- add global Kick discovery;
- remove registry fallback;
- claim Kick parity;
- claim directory coverage.

## Example

```bash
cd ~/viewloom
node workers/collector-kick/scripts/probe-kick-live-candidates.mjs
cd ~/viewloom/workers/collector-kick
cat generated/kick-live-priority-boost.sql
npx wrangler d1 execute vl_kick_hot --remote --file generated/kick-live-priority-boost.sql
```
