# PR-176 Kick registry migration and import runbook

## Summary

This PR adds the safe operator runbook for applying the `kick_channels` migration and importing initial Kick seed candidates.

## What was added

- `docs/kick-registry-migration-import-runbook.md`

## Why

The repository already contains:

- `workers/collector-kick/migrations/0001_create_kick_channels.sql`
- `workers/collector-kick/scripts/generate-kick-seed-import-sql.mjs`
- `docs/kick-channel-registry-schema.md`
- `docs/kick-registry-seed-import-plan.md`
- `docs/kick-candidate-expansion-plan.md`

This PR defines the execution order, verification queries, failure handling, and coverage-mode guard before any production D1 command is run.

## Key rules

- Do not run production commands unless the operator intentionally chooses to do so.
- Do not switch `coverageMode` from `seed-list` to `registry` after seed import alone.
- Do not describe Kick as Twitch parity or directory coverage.
- The collector must read from `kick_channels` and write a valid registry-selected production snapshot before `coverageMode='registry'` is valid.

## Scope

Docs only.

This PR does not:

- execute migration;
- execute seed import;
- change collector runtime;
- change API status behavior;
- add discovery;
- claim Kick is fixed;
- claim Kick is Twitch parity.
